# ASR 语料处理与模型打包系统深度技术手册

## 1. 系统宏观架构
本系统是一个高度集成的 ASR（自动语音识别）离线资源构建平台，其核心任务是将非结构化的原始语料转化为具备声学、词典和语言学约束的二进制 `.bin` 模型。

### 核心三剑客：
*   **`corpus_process.py`**：语料预处理引擎。负责清洗、分词、扩充及 OOV（未登录词）提取。
*   **`net_maker.py`**：语法网络构建器。负责 G Package（正则模式）的图结构生成及嵌套槽位展开。
*   **`corpus_process_package.py`**：总调度与打包器。负责驱动 N-gram 训练工具及 WFST 序列化二进制工具。

## 2. 语料预处理 (Corpus Processing) —— 深度解析
预处理是整个系统的“数据洗髓池”，其质量直接决定了识别率。

### 2.1 输入层：多模态兼容
系统通过 `get_corpus_process` 函数根据语种和路径初始化。
*   **Excel 模式**：解析 `sent`（标准句）、`shuofa`（模板）和 `<slot>`（槽位）。
*   **文本模式**：直接读取文件夹下的 `.txt` 文件，每行作为一个独立的语料单元。

### 2.2 核心步骤 1：说法与槽位扩展 (Expansion)
这是处理业务逻辑最关键的一步。
*   **逻辑原理**：通过“笛卡尔积”算法，将 `shuofa` 中的模板与 `slot` 中的词表进行组合。
*   **示例**：
    *   模板：`我想听 <singer> 的歌`
    *   槽位词：`周杰伦`, `王菲`
    *   **结果**：生成 `我想听周杰伦的歌`、`我想听王菲的歌`。
*   **采样控制**：通过 `--expand_argument`（如 `poi-20`）限制每个槽位的采样上限，防止语料库爆炸。

### 2.3 核心步骤 2：文本规整 (Normalization)
*   **基础规整**：全角转半角、大写转小写、删除非法字符（如表情符号、特殊标点）。
*   **数字转写**：处理语料中的 `0x`（十六进制）或普通数字，将其转换为发音词典可识别的字符形式。

### 2.4 核心步骤 3：多语种分词 (Tokenization)
系统针对不同语言特征集成了专门的处理引擎：
*   **泰语 (Thai)**：调用 `pythainlp.tokenize.subword_tokenize`。由于泰语没有空格，子词切分对于后续的词典匹配至关重要。
*   **日语 (Japanese)**：调用 `spacy` (`ja_core_news_lg`)。利用形态学分析进行精准分词，并过滤掉无意义的助词。
*   **其他语种**：默认按空格切分。

### 2.5 核心步骤 4：OOV 提取与词典过滤
*   **匹配**：将规整分词后的每一个词与 `new_dict` 匹配。
*   **提取**：凡是词典中不存在的词，统一收集到 `aaa_oov_base_dict` 中。
*   **意义**：这个 OOV 文件是后续 G2P（发音预测）步骤的唯一输入源，确保所有新词都能获得准确的发音。

## 3. G Package 模式与嵌套槽位逻辑
当开启 `-G` 参数时，系统进入“正则/语法图”打包模式，不计算概率，只讲规约。

### 3.1 G 网络构建逻辑
*   **数学本质**：使用 `net_maker.py` 构建一个 WFST（加权有限状态转换器）。
*   **权重分配**：通过 `-G 1-6` 中的 `6` 给所有有效路径赋权重（Cost），使其在识别时优于通用语料。

### 3.2 大槽包小槽（嵌套槽位）实现
这是通过 `net_expand` 函数中的 **宽度优先动态展开算法** 实现的：
1.  **扫描**：在主图中发现 `<大槽>` 标记。
2.  **定位**：从 `allslot_net` 中获取大槽对应的子图。
3.  **缝合**：将子图插入主图。
4.  **递归检查**：在缝合过程中，如果发现子图中还嵌套了 `<小槽>`，则将该小槽加入任务队列。
5.  **循环**：持续执行直到队列清空。
*   **优势**：支持无限层级的语法嵌套，极大简化了复杂业务逻辑的维护。

## 4. HMM 列表 (HMM List) 与物理映射
`hmmlist` 是“声学模型”与“语言网络”之间的唯一翻译官。

### 4.1 核心区别：Phone vs Triphone
*   **UBCTC (is_yun=0)**：使用 **Triphone（三音素）**。
    *   格式：`L-X+R`（如 `k-ae+t`）。
    *   特点：显式建模上下文，HMM 列表通常有几千行。
*   **RNN-T (is_yun=1/2)**：使用 **Phone（单音素）**。
    *   格式：单一字符（如 `ae`）。
    *   特点：隐式建模上下文（依靠神经网络），HMM 列表通常只有几十行。

### 4.2 映射机制
*   **索引即 ID**：`hmmlist` 的**行号**直接对应神经网络输出层的 **PDF-ID**（或音素 ID）。
*   **打包用途**：打包工具（如 `package_speed_mhwu`）读取该文件，并在 WFST 图的每一个弧（Arc）上标记对应的行号 ID。
*   **解码用途**：手机端识别时，神经网络吐出的数字（ID）通过查阅这份列表，变回人类可读的发音标签。

## 5. 二进制工具箱 (`bin/`) 功能速查

| 工具名称 | 功能描述 | 适用场景 |
| :--- | :--- | :--- |
| **`ngram-count`** | 统计词频概率，生成 3-gram LM | 传统 N-gram 模式 |
| **`package_speed_mhwu`** | 通用打包工具，执行 H-C-L-G 融合 | 离线端侧模型（泰、日、韩等） |
| **`esr_package_...`** | 基础版打包工具 | 语种 0 (En) + 传统离线架构 |
| **`package_ed_sp_v2`** | RNNT / Encoder-Decoder 专用打包 | 新型端到端架构 |
| **`package_add_blank`** | 强行在网络中插入 `blank` 路径 | CTC 架构模型 |
| **`fst_bin/fstcompile`等** | 对 WFST 进行数学优化 | G 模式下的图压缩 |

## 6. 自动化调用示例 (集成版)
在 `asr_mlg` 根目录下执行（已实现路径和配置的全自动推导）：

```bash
# 启动集成流水线
python pipeline/pipeline_executor.py -j pipeline/job.yaml
```

### `job.yaml` 极简示例：
```yaml
tasks:
  - msg: "thai_car_project"
    l: 10                   # 泰语
    -G: "1-6"               # 开启 G 网络打包
    excel_corpus_path: "project_corpus/thai"
    is_yun: 0               # 端侧打包
    enable_g2p: true        # 自动处理新词发音
```
